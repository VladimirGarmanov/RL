# RL Car Racing — машинка учится гонять по F1-трассе

2D-гоночный симулятор с реалистичной физикой шин. Машинка обучается ехать
по трассе через обучение с подкреплением (Reinforcement Learning, PPO).

![Python](https://img.shields.io/badge/Python-3.10+-blue) ![pygame](https://img.shields.io/badge/pygame-2.x-green) ![SB3](https://img.shields.io/badge/stable--baselines3-PPO-orange)

## Если вы не знакомы с RL — кратко

Агент (нейросеть) не знает правил вождения. На каждом шаге он «видит»
наблюдение (13 чисел: лучи-дальномеры, скорость, компас на следующий
чекпоинт), жмёт одну из 7 кнопок и получает награду — число «насколько
хорошо». Алгоритм PPO миллионы раз повторяет цикл «попробовал — получил
награду — подстроил веса сети», постепенно превращая случайные дёрганья
в аккуратное прохождение поворотов. Никакого «если стена слева — руль
вправо» в коде нет: всё поведение выучивается из награды.

## Быстрый старт

```bash
# Установи зависимости (из корня репо)
pip install -r requirements.txt

# Ручной режим — поездить самому (та же физика, что у агента)
python main.py

# Обучение агента (2M шагов; Ctrl+C — остановить, модель сохранится)
python train.py

# Посмотреть обученную модель
python play_trained.py
```

**Управление в ручном режиме:**
- `↑` — газ
- `↓` — тормоз / задний ход
- `←` / `→` — поворот
- `R` — сброс позиции
- `Esc` — выход

## Как устроено

### Физика (car.py): окружность трения

Ключевая идея из реальной физики: шина может передать на асфальт
ограниченную суммарную силу. Она делится между тягой/торможением
(продольная) и удержанием в повороте (боковая):
`sqrt(продольная² + боковая²) ≤ бюджет сцепления`. Следствие: в быстром
повороте тяга и тормоза слабеют, а перебор скорости даёт занос (машинка
синеет — визуальная индикация скольжения). Именно поэтому агенту
приходится учиться тормозить ПЕРЕД поворотом, как настоящему гонщику.

### Мир и трассы (track.py)

- Мир **3000×2000 px**, камера следует за машинкой, мини-карта в углу.
- Вариант 0 — F1-трасса по мотивам Спа-Франкоршам (шпилька, S-образная
  связка, длинные прямые, шиканы), 39 контрольных точек, сглаженных
  сплайном Катмулла-Рома.
- Вариант 1 — «восьмёрка» с перекрёстком (кривая Лиссажу).
- Варианты 2–31 — случайные трассы (ряд Фурье в полярных координатах —
  гарантированно без самопересечений). Каждый эпизод обучения — случайная
  трасса из пула, поэтому агент учится ездить «вообще», а не зубрит одну.

### Наблюдения (13 чисел, все в [-1, 1])

| # | Что |
|---|-----|
| 0–6 | 7 лучей-дальномеров до края дороги (углы -90°…+90°) |
| 7 | продольная скорость (нормированная) |
| 8 | боковая скорость / дрейф (нормированная) |
| 9–10 | sin/cos курса машинки |
| 11–12 | sin/cos угла на следующий чекпоинт («компас») |

### Действия (7 дискретных кнопок)

`ничего` / `газ` / `газ+влево` / `газ+вправо` / `тормоз` / `влево` / `вправо`

### Награда (car_env.py)

Главный принцип: платить только за настоящий прогресс к следующему
чекпоинту и жёстко закрывать все лазейки (каждая из них была найдена
агентом в ранних версиях и залатана):

| Событие | Reward |
|---------|--------|
| Приближение к чекпоинту (по направлению трассы) | +0.05/px |
| Удаление от чекпоинта | −0.14/px |
| Чекпоинт (только по порядку и только на дороге) | +5.0 |
| Круг завершён | +20.0 |
| Выезд с дороги (любая из 6 точек кузова) | −40 − 2×скорость, конец эпизода |
| Пропуск чекпоинта (подъехал ближе 95px и уехал дальше 115px) | −25, конец эпизода |
| Скорость выше безопасной перед поворотом | квадратичный штраф |
| Боковое скольжение сверх порога | штраф |
| Езда у края / поперёк / против хода трассы | штраф |
| Вращение и стояние на месте | штраф |

## Структура

```
car_racing/
├── main.py          — ручной режим (те же физика и трасса, что у агента)
├── train.py         — обучение PPO (Stable-Baselines3)
├── play_trained.py  — просмотр обученной модели
├── car_env.py       — Gymnasium-среда: наблюдения, награда, чекпоинты
├── car.py           — физика машинки (окружность трения) + лучи-сенсоры
├── track.py         — F1-трасса, восьмёрка, генератор случайных трасс
├── utils.py         — шрифты и HUD (+ обход бага pygame.font на Python 3.14)
└── models/          — сохранённые модели (не в git)
    └── ppo_car_racing.zip
```

## Настройка обучения

В `train.py`:
```python
TOTAL_TIMESTEPS = 2_000_000  # шагов обучения (200k — базовое, 2M — уверенное)
RESUME_TRAINING = False      # True = дообучать сохранённую модель
```

Гиперпараметры PPO — словарь `PPO_PARAMS` там же. Веса награды —
константы в начале `car_env.py` (каждая подписана).
