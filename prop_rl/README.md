# prop_rl — 13" воздушные винты, спроектированные RL-агентом

## Что это за проект

Обучение с подкреплением (Reinforcement Learning, RL) обычно применяют к
играм, но здесь оно решает **инженерную задачу**: спроектировать воздушный
винт (пропеллер) для дрона. Агент не видит экрана и не жмёт кнопки — он
выдаёт 9 чисел (геометрию винта и обороты), а физическая модель говорит,
насколько такой винт хорош. Это одношаговая задача (в RL-терминах —
«бандит»): одно действие = один готовый дизайн.

Винт вписан в цилиндр диаметром 13 дюймов (330 мм). Агент подбирает
**хорду (ширину лопасти) в 4 сечениях, крутку (угол установки) в 4
сечениях и рабочие обороты**. Две цели — два разных винта:

- **quiet** (`prop_13in.stl`): минимум шума при взлётной тяге >= 1.2 кгс;
- **thrust** (`prop_13in_thrust.stl`): максимум тяги при мощности <= 300 Вт.

Сравнение форм — `preview_prop_both.png`.

## Как всё устроено (конвейер из трёх файлов)

```
prop_train.py  --учит агента-->  best_prop.json  --геометрия-->  prop_stl.py  -->  prop_13in.stl
     |                                                                                (на 3D-принтер)
     +--- на каждом шаге спрашивает физику у prop_env.py
```

1. **`prop_env.py`** — «среда»: превращает 9 чисел агента в геометрию
   винта, считает тягу/мощность/шум (физика BEMT) и возвращает награду.
2. **`prop_train.py`** — «агент»: алгоритм REINFORCE с гауссовой политикой.
   600 итераций по 96 случайных дизайнов; распределение постепенно
   стягивается к хорошим вариантам. Лучший допустимый дизайн -> JSON.
3. **`prop_stl.py`** — «чертёжник»: строит из JSON печатаемую 3D-модель
   (профиль NACA 4412, лофт по сечениям, ступица) и проверяет её
   водонепроницаемость.

## Запуск

```bash
python3 prop_train.py            # тихий  -> best_prop.json
python3 prop_stl.py              #        -> prop_13in.stl

python3 prop_train.py thrust     # тяговый -> best_prop_thrust.json
python3 prop_stl.py best_prop_thrust.json   # -> prop_13in_thrust.stl
```

Зависимости: `numpy`, `torch` (см. `requirements.txt` в корне репозитория).
Обучение занимает несколько минут на CPU.

## Тихий против тягового

| Параметр | Тихий | Тяговый |
|---|---|---|
| Тяга | 1.23 кгс | **2.07 кгс** |
| Обороты | 3488 | 6443 |
| Мощность | 161 Вт | 300 Вт (упёрся в лимит) |
| Шум (прокси) | 49 дБ | 67 дБ (в ~3.5 раза громче на слух) |
| V конца лопасти | 60 м/с | 111 м/с |
| Форма | «вёсла» до самого конца | широкий комель, **узкий конец** (15 мм), крутка 29°→6° |
| Эффективность | 7.6 г/Вт | 6.9 г/Вт |

Агент сам переоткрыл оба классических рецепта авиастроения: для тишины —
тянуть обороты вниз и добирать площадью; для тяги — крутить до лимита
мощности и **сужать конец лопасти** (там максимальный скоростной напор,
узкая хорда экономит профильное сопротивление, которое иначе съедает ватты).

## Физика (prop_env.py)

- **BEMT** (blade element + momentum theory) — классический метод расчёта
  винтов: лопасть режется на 30 кольцевых элементов; для каждого
  итерационно согласуются сила профиля (Cl, Cd Кларк-Y-подобного профиля
  с учётом срыва потока) и индуктивная скорость из баланса импульса,
  с концевыми потерями Прандтля. Модель откалибрована: для геометрии
  APC 13x4.5 на 5000 об/мин даёт 1.2 кгс / 141 Вт — совпадает с реальными
  обмерами.
- **Шум** — инженерный прокси (упрощённая оценка с верным направлением):
  широкополосный (вихревой) член растёт как скорость на 75% радиуса
  **в шестой степени** плюс член нагрузки. Абсолютные дБ условны, но
  градиент физически верный: главный путь к тишине — снижать скорость
  конца лопасти, компенсируя площадью и круткой.

## Что нашёл агент (цель quiet)

| Параметр | Обычный 13x4.5 | RL-винт |
|---|---|---|
| Обороты для 1.23 кгс | ~5000 | **3488** |
| Скорость конца лопасти | 86 м/с | **60 м/с** |
| Шум (прокси) | ~58 дБ | **49 дБ** (~ вдвое тише на слух) |
| Мощность | 141 Вт | 161 Вт |
| Хорда | ~28 мм | до 42 мм («вёсла») |

Классический компромисс тихих винтов: −9 дБ ценой +14% мощности.
Крутка почти постоянная ~22°: угол атаки по всей лопасти держится у
9–11° — близко к максимуму аэродинамического качества профиля.

## Печать и безопасность

- `prop_13in.stl`: 330 x 58 x 18 мм, ~50 г PLA, ступица d32 мм, вал 5 мм.
  На стол 250x250 влезает по диагонали.
- Печатать плашмя, поддержки под лопасти (крутка отрывает кромки от
  стола), 100% заполнение, слой 0.15 мм. Лучше **PETG или нейлон** — PLA
  хрупкий и ползёт от нагрева мотора.
- **Осторожно**: печатный винт слабее литого — слои могут расслоиться от
  центробежной нагрузки. Первый запуск — закреплённо, за преградой, без
  людей в плоскости вращения. Не превышать расчётные ~3500 об/мин
  (запас по центробежке у PLA/PETG на 6000+ уже сомнительный).
- Балансировать после печати (скотч на лёгкую лопасть / шлифовка тяжёлой).

## Честные оговорки

- Прокси шума не знает про шум концевого вихря, поэтому агент оставил
  конец лопасти широким (ему «бесплатно»); реальные тихие винты сужают
  и скругляют кончик — можно дошлифовать вручную.
- BEMT — статика (висение/взлёт); в горизонтальном полёте оптимум другой.
- Профиль один по всему радиусу (NACA 4412); настоящие винты меняют
  профиль к комлю.

## Файлы

- `prop_env.py` — физика и награда: BEMT (`bemt`), модель шума
  (`noise_proxy`), распаковка действия (`action_to_params`,
  `interp_blade`), расчёт награды (`evaluate_single`, `evaluate`)
- `prop_train.py` — обучение: REINFORCE-бандит, 600 итераций x 96 дизайнов
- `prop_stl.py` — 3D-модель: профиль NACA 4412 (`naca4412`), лофт лопасти
  (`section_3d`, `blade_mesh`), ступица (`hub_mesh`), проверки геометрии
  (`shell_volume`, `ensure_outward`, `check_shell`), запись STL (`write_stl`)
- `best_prop.json` / `best_prop_thrust.json` — найденные параметры и режим
- `prop_13in.stl`, `preview_prop.png` — модель и превью
