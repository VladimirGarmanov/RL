# Быстрый старт за 5 минут

## 1. Установка (первый раз)

```bash
cd centipede
pip install -r ../requirements.txt
```

## 2. Обучение — МАКСИМУМ ПЛАВНОСТИ (одна команда)

```bash
python train_overnight.py
```

Это запустит обучение одной универсальной сети на **180 млн шагов** (~80–100 часов на CPU).

Робот научится:
- ✅ Идти вперёд / назад (стрелки ↑ ↓) — **ОЧЕНЬ плавно**
- ✅ Поворачивать влево / вправо (← →) — без рывков
- ✅ **Активно балансировать даже когда стоит** на месте (микродвижения)
- ✅ Слушаться стрелок мгновенно, но без конвульсий
- ✅ Не падать никогда

### Время обучения (рекомендации):

| Процессор | Время |
|-----------|-------|
| MacBook M1/M2 (8 ядер) | 80–100 часов (~4 дня) |
| Intel i9 (8 ядер, 3.6 ГГц) | 100–120 часов (~5 дней) |
| AWS t3.2xlarge (8 vCPU) | 80–100 часов + пинг (~4–5 дней) |
| GPU (RTX 3080) | 8–10 часов |

**Рекомендуется:** запустить в `screen` или `tmux`, чтобы не завязывать терминал:

```bash
# Запустить обучение в фоновом режиме (не завязывает терминал)
screen -S centipede_train
python train_overnight.py
# Ctrl+A, D  — отсоединиться от сессии
# screen -r centipede_train  — вернуться к сессии

# Или в tmux:
tmux new-session -d -s centipede 'python train_overnight.py'
tmux attach -t centipede  # просмотр логов
```

## 3. После обучения (через ~100 часов) — смотреть результат

```bash
# Управление стрелочками в окне — максимально плавная ходьба
python play.py

# Или робот сам ездит по программе
python play.py --auto
```

### Как использовать:
- **Стрелки ↑ ↓ ← →** — ехать в направлении
- **Пробел** — стоп
- **R** — рестарт эпизода
- **Мышь + ЛКМ** — вращать камеру
- **Колесо мыши** — зум
- **F** — включить/выключить следование камеры за роботом
- **Home** — сброс камеры
- **Esc** — выход

## 4. Если обучение зависло / нужно продолжить

```bash
python train_overnight.py --resume
```

## 5. С Docker (если проблемы с зависимостями)

```bash
docker build -t centipede .
docker run -v $(pwd)/models:/app/centipede/models centipede train_overnight.py
```

Или (через compose, логи в файл):
```bash
docker-compose build
docker-compose up train
tail -f train_docker.log  # смотреть логи в реальном времени
```

## Куда смотреть при обучении

Ищите в логах строку `ep_len_mean`:
- **< 100** — робот сразу падает (что-то не так)
- **100–500** — кое-что работает, но ещё шатается
- **> 700** — хорошо, робот ходит плавно и не падает

Должно расти со временем. Если не растёт 1+ час — остановите (`Ctrl+C`), проверьте логи.

## Что потом?

На реальный робот переносить нужно:
1. `models/ppo_centipede_final.zip` — веса политики
2. `models/vecnormalize_final.pkl` — статистика нормализации наблюдений
3. Код препроцессинга из `centipede_env.py:_get_obs()` и нормализации
4. НЧ-фильтр действий (`action_filter_alpha = 0.3`)

Это займёт ~0.77 МБ памяти на микроконтроллере (Teensy 4.1 или ESP32-S3).
