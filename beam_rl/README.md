# beam_rl — балка, спроектированная RL-агентом, под 3D-печать

## Что это за проект

Обучение с подкреплением (Reinforcement Learning, RL) здесь решает не
игру, а **инженерную задачу**: спроектировать балку, которая держит
40 кг (400 Н) в центре пролёта 150 мм, при минимальной массе PLA-пластика.
Агент выдаёт числа (параметры дизайна), физическая модель (сопромат/МКЭ)
считает прочность и возвращает награду, агент учится выдавать числа лучше.

Два независимых варианта — от простого к сложному:

1. **Двутавр** (`beam.stl`) — параметрическая оптимизация: агент одним
   действием подбирает 4 размера сечения. Одношаговая RL-задача («бандит»).
2. **Органическая** (`beam_organic.stl`) — топологическая оптимизация:
   агент за 25 шагов «рассасывает» слабонагруженный материал из сплошной
   балки, как кость по закону Вольфа. Настоящая многошаговая RL-задача
   с отложенной наградой.

Подробная документация каждой функции — в [DOCS.md](DOCS.md).

## Вариант 1: параметрический двутавр

Двутавр — сечение в форме буквы «I»: две горизонтальные полки, соединённые
вертикальной стенкой. Самая эффективная классическая форма балки: материал
разнесён от нейтральной оси туда, где он работает.

Конвейер:

```
train.py --REINFORCE--> best_beam.json --геометрия--> make_stl.py --> beam.stl
    |                                                                (на принтер)
    +--- физику (напряжение, прогиб, массу) считает beam_env.py
```

### Запуск

```bash
python3 train.py     # обучение, ~10 сек, сохраняет best_beam.json
python3 make_stl.py  # генерирует beam.stl (с проверкой водонепроницаемости)
```

### Результат

Двутавр 150 × 31.5 × 15.5 мм, масса ~18 г (100% заполнение):

| Параметр | Значение |
|---|---|
| Высота H | 31.5 мм |
| Ширина полки B | 15.5 мм |
| Толщина полки / стенки | 1.6 мм (4 периметра сопла 0.4) |
| Напряжение при 40 кг | 16.7 МПа (предел PLA 50 МПа → запас ×3) |
| Прогиб при 40 кг | 0.6 мм |

Агент упёрся в оба ограничения сразу (минимальная печатная толщина и
допускаемое напряжение) — это признак действительно оптимального решения.

## Вариант 2: органическая балка (топологическая оптимизация)

Конвейер:

```
topo_train.py --REINFORCE--> best_topology.npy --marching squares--> topo_stl.py --> beam_organic.stl
      |                       (карта 0/1: где материал)
      +--- физику считает МКЭ-решатель в topo_env.py
```

### Запуск

```bash
python3 topo_train.py   # ~2 мин: REINFORCE учится стратегии резьбы
python3 topo_stl.py     # marching squares -> beam_organic.stl + МКЭ-проверка
```

### Как это работает

Половина балки (задача симметрична — вторая половина зеркальна)
моделируется МКЭ-сеткой 60×24 из квадратных клеток 1.25 мм (плоское
напряжённое состояние, билинейные четырёхузловые элементы). МКЭ — метод
конечных элементов: деталь режется на клетки, для каждой известна связь
«силы ↔ перемещения», из них собирается система уравнений, решение
которой говорит, как деталь деформируется под грузом.

Эпизод — 25 шагов; на каждом агент выбирает **долю удаляемого материала**
и **температуру** (насколько жадно резать именно самые ненагруженные
клетки). Клетки удаляются с вероятностью, растущей при малой энергии
деформации, — это буквально ремоделирование кости. Награда приходит
только в конце эпизода — жёсткость итоговой структуры при целевых 45%
материала.

### Результат

`preview_organic.png`: 150 × 30 × 12 мм, **33 г** (сплошная — 67 г),
арка + ветвящиеся подкосы. При 40 кг в центре: sigma_max 17 МПа (предел
PLA 50 МПа, запас ×2.9), прогиб 0.7 мм. Жёсткость — 46% сплошной при
45% массы.

## Как печатать (важно для прочности!)

### Двутавр (beam.stl)

STL уже лежит в правильной ориентации — **на боку**, профилем «H».
Слои идут вдоль длины балки, поэтому нагрузка не расслаивает деталь.
Стенка печатается мостом (~28 мм) между полками — любой принтер справится.

- Материал: **PLA** (или PETG — чуть менее жёсткий, но не хрупкий)
- Периметры: **4+** (стенки толщиной 1.6 мм станут монолитными)
- Заполнение: расчёт сделан для сплошного сечения — ставь **100%**
  (при 4 периметрах заполнения внутри почти не остаётся, разница в граммах мала)
- Не переворачивать и не ставить вертикально: вертикальная печать
  ослабит балку на порядок (нагрузка пойдёт поперёк слоёв)

**При использовании** поверни балку профилем «I»: высокая сторона
(31.5 мм) вертикально, груз кладётся на верхнюю полку, опоры под края.
Если нагружать её лёжа (как печаталась) — жёсткость упадёт в разы:
работает слабая ось.

### Органическая (beam_organic.stl)

Балка лежит плашмя, каждый слой одинаковый — **ни нависаний, ни
поддержек, ни мостов**. PLA/PETG, 4+ периметра, 100% заполнение (стенки
тонкие, заполнения почти не остаётся). Нагружать в плоскости узора:
опоры под нижние углы, груз сверху в центр, на плоскую площадку.

## Файлы

| Файл | Роль |
|---|---|
| `beam_env.py` | физика двутавра (изгиб, прогиб, масса) + gymnasium-среда |
| `train.py` | REINFORCE: ищет самое лёгкое допустимое сечение |
| `make_stl.py` | генерация водонепроницаемого STL двутавра (ушная триангуляция) |
| `best_beam.json` | найденные параметры двутавра |
| `beam.stl` | готовая модель двутавра для слайсера |
| `topo_env.py` | МКЭ-решатель + среда «резьбы» (закон Вольфа) |
| `topo_train.py` | REINFORCE для органической балки |
| `topo_stl.py` | marching squares → `beam_organic.stl` (+ проверка прочности) |
| `best_topology.npy` | карта материала лучшей органической структуры |
| `DOCS.md` | подробная документация каждой функции |
