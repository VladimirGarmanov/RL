# beam_rl — подробная документация каждого файла и функции

## Мини-глоссарий (если термины незнакомы)

- **RL (Reinforcement Learning, обучение с подкреплением)** — агент выдаёт
  действия, среда возвращает награду (число «насколько хорошо»), агент
  подстраивается, чтобы награда росла.
- **Политика** — правило выбора действия. Здесь — нормальное (гауссово)
  распределение: у каждого действия обучаемое среднее и разброс.
- **REINFORCE** — простейший алгоритм RL: действия с наградой выше средней
  делаем вероятнее, ниже средней — менее вероятными.
- **Бандит** — одношаговая задача: одно действие = готовый результат,
  без последовательности состояний.
- **Эпизод** — последовательность шагов от reset() до конца (у органической
  балки — 25 шагов резьбы).
- **Advantage / baseline** — награда минус средняя по батчу: важно не
  абсолютное значение, а «лучше ли остальных»; сильно снижает шум обучения.
- **МКЭ (метод конечных элементов, FEM)** — деталь режется на клетки,
  для каждой известна связь «силы ↔ перемещения», из них собирается система
  уравнений K·u = F; решение u — деформация детали под грузом.
- **Податливость (compliance)** — F·u: чем меньше, тем жёстче структура.
- **Напряжение (σ, МПа)** — «плотность силы» в материале; выше предела
  прочности — деталь ломается.
- **Прогиб** — насколько балка провисает под грузом.
- **Watertight (водонепроницаемый) меш** — замкнутая 3D-оболочка без дыр:
  обязательное условие для 3D-печати.

Проект содержит два независимых пайплайна:

- **Параметрический двутавр**: `beam_env.py` → `train.py` → `best_beam.json` → `make_stl.py` → `beam.stl`
- **Органическая балка** (топологическая оптимизация): `topo_env.py` → `topo_train.py` → `best_topology.npy` → `topo_stl.py` → `beam_organic.stl`

Общее техзадание: балка на двух опорах, пролёт 150 мм, груз 400 Н (~40 кг)
в центре, материал PLA (E = 3300 МПа, предел прочности 50 МПа, плотность
1.24 г/см³), запас прочности ×3 → допускаемое напряжение ~16.7 МПа.

## beam_env.py — физика и среда для параметрического двутавра

Секция констант: техзадание (`SPAN_MM`, `DESIGN_LOAD_N`), материал
(`E_MPA`, `SIGMA_ULT_MPA`, `SAFETY_FACTOR`, `SIGMA_ALLOW_MPA`,
`DENSITY_G_MM3`), лимит прогиба и границы 4 параметров сечения
(`PARAM_LOW`, `PARAM_HIGH`): H — высота балки, B — ширина полки,
tf — толщина полки, tw — толщина стенки. Нижняя граница толщин 1.6 мм =
4 периметра сопла 0.4 мм (тоньше печатать нельзя без потери монолитности).

### `action_to_params(action) -> params`
Переводит «сырое» действие агента (любые вещественные числа, батчами
формы `(..., 4)`) в физические размеры сечения:
1. `tanh` сжимает действие в [-1, 1] — политика может выдавать что угодно,
   границы не нарушатся;
2. линейное отображение в диапазон `[PARAM_LOW, PARAM_HIGH]`;
3. два геометрических клампа: `tf ≤ (H-4)/2` (между полками должно
   остаться ≥4 мм стенки, иначе двутавр вырождается) и `tw ≤ B`
   (стенка не шире полки).

### `analyze(params) -> (sigma, deflection, mass_g)`
Сопромат трёхточечного изгиба для двутавра, всё батчами numpy:
- момент инерции `I = (B·H³ − (B−tw)·(H−2tf)³) / 12` — прямоугольник
  минус два «выреза» по бокам стенки;
- момент сопротивления `W = I / (H/2)`;
- максимальный изгибающий момент `M = P·L/4` (нагрузка в центре
  двухопорной балки);
- напряжение в крайнем волокне `σ = M / W` (МПа);
- прогиб в центре `δ = P·L³ / (48·E·I)` (классическая формула);
- масса: площадь сечения `2·B·tf + tw·(H−2tf)` × длина × плотность.

### `evaluate(params) -> (reward, feasible, sigma, deflection, mass_g)`
Функция награды поверх `analyze`:
- `over_stress = max(0, σ/σ_доп − 1)`, `over_defl = max(0, δ/δ_max − 1)` —
  относительные превышения ограничений (0, если всё в норме);
- `reward = 5 − масса/20 − 40·over_stress − 40·over_defl` — минимизируем
  массу, штрафы делают нарушение ограничений всегда невыгодным;
- `feasible` — булев флаг «оба ограничения выполнены».

### `class BeamEnv(gym.Env)`
Обёртка gymnasium для совместимости с экосистемой RL. Среда одношаговая
(contextual bandit): наблюдение фиктивное (нулевой вектор), действие —
Box(-1, 1, shape=(4,)), эпизод заканчивается после первого `step`.
- `reset()` — возвращает нулевое наблюдение;
- `step(action)` — считает награду через `evaluate`, кладёт в `info`
  все физические величины, `terminated=True`.

## train.py — REINFORCE для двутавра (скрипт, функций нет)

Гауссова политика без нейросети: обучаемые параметры — вектор средних
`mu (4,)` и лог-СКО `log_std (4,)` (состояния нет, задача одношаговая).
Цикл (1500 итераций × батч 256):
1. сэмплируются 256 действий из `Normal(mu, std)`, `std` клампится в
   [0.02, 2.0] (не даёт политике ни схлопнуться, ни разлететься);
2. награды считаются векторизованно через `evaluate`;
3. лучшее **допустимое** решение батча сравнивается с рекордом и
   запоминается (RL шумит, а нам нужен конкретный лучший дизайн);
4. градиент REINFORCE: `loss = −E[(r − baseline)·log π(a)]`, где
   baseline — среднее награды по батчу (снижает дисперсию), плюс
   маленький энтропийный бонус `1e-3` против преждевременного сжатия;
5. шаг Adam (lr = 0.03).

В конце: детерминированный прогон политики (среднее `mu`), к рекорду
дописываются фактический запас прочности и параметры техзадания, всё
сохраняется в `best_beam.json`.

## make_stl.py — генерация STL двутавра

### `i_profile(H, B, tf, tw) -> (12, 2) ndarray`
12 вершин контура двутавра в плоскости профиля (a — направление ширины,
b — высоты), обход против часовой стрелки, начиная с левого нижнего угла.

### `triangulate(poly) -> [(i, j, k), ...]`
Ушная триангуляция (ear clipping) простого многоугольника без дыр,
обход CCW. На каждом шаге ищется «ухо» — выпуклая вершина, треугольник
которой не содержит других вершин, — отрезается, и так пока не останется
3 вершины. Вспомогательные замыкания: `cross` — z-компонента векторного
произведения (знак = выпуклость), `point_in_tri` — принадлежность точки
треугольнику через три знака. Используется и в `topo_stl.py`.

### `build_mesh(H, B, tf, tw, L) -> (N, 3, 3) ndarray`
Строит замкнутый меш балки **сразу в ориентации для печати**: X — длина,
Y — высота профиля H, Z — ширина полки B (вертикаль стола). То есть
балка лежит на боку H-профилем.
- вложенная `v(x, p)` — переводит 2D-точку профиля в 3D
  (a→Z с подъёмом на B/2, чтобы низ был на Z=0; b→Y);
- торцы x=0 и x=L: триангуляция профиля; отображение (a,b)→(Z,Y) меняет
  хиральность, поэтому прямой порядок вершин закрывает торец x=0
  (нормаль −X), обратный — x=L (нормаль +X);
- боковые стенки: каждое ребро контура даёт прямоугольник из двух
  треугольников между торцами.

### `check_mesh(tris, expected_volume) -> volume`
Три проверки корректности меша:
1. водонепроницаемость: каждое неориентированное ребро встречается ровно
   у двух треугольников (словарь рёбер по округлённым координатам);
2. объём по теореме о дивергенции `V = Σ dot(v0, cross(v1, v2))/6 > 0`
   (положительный ⇒ нормали наружу);
3. объём совпадает с аналитическим (площадь сечения × длина) с точностью
   1e-6 — ловит вывернутые крышки и потерянные треугольники.

### `write_stl(path, tris)`
Бинарный STL: 80-байтовый заголовок, число треугольников (uint32), затем
на каждый треугольник — нормаль (считается из векторного произведения и
нормируется), 3 вершины по 3 float32 и 2 байта атрибутов.

### Блок `__main__`
Читает `best_beam.json`, строит меш, проверяет через `check_mesh`
(ожидаемый объём — аналитический), пишет `beam.stl`, печатает габариты,
массу и сводку прочности.

## topo_env.py — МКЭ и среда «резьбы» для органической балки

Моделируется **половина** балки (симметрия относительно середины
пролёта — это вдвое меньше степеней свободы): сетка `NX×NY = 60×24`
квадратных элементов по 1.25 мм, плоское напряжённое состояние толщиной
`THICK_MM = 12` мм. `TARGET_VOL` — целевая доля оставляемого материала.
`E_VOID = 1e-6·E0` — «пустые» клетки не удаляются из модели, а получают
исчезающе малую жёсткость (soft-kill SIMP-подход), чтобы матрица
жёсткости не вырождалась.

### `E_MAT_UNIT() -> (3, 3)`
Матрица упругости плоского напряжённого состояния для E=1:
`1/(1−ν²) · [[1, ν, 0], [ν, 1, 0], [0, 0, (1−ν)/2]]`. Умножается на
фактический модуль элемента при сборке и при расчёте напряжений.

### `b_matrix(xi, eta) -> (3, 8)`
B-матрица (деформации = B · перемещения узлов) билинейного
четырёхузлового элемента в точке (ξ, η) локальных координат [-1,1]².
Производные функций формы умножаются на `2/ELEM_MM` — обратный якобиан
квадратного элемента. Раскладка строк: εx, εy, γxy; столбцы чередуют
ux, uy четырёх узлов (CCW от левого нижнего).

### `element_stiffness() -> (8, 8)`
Матрица жёсткости элемента гауссовой квадратурой 2×2:
`K = Σ Bᵀ·D·B · detJ · t` по четырём точкам ±1/√3. Для квадратного
элемента результат не зависит от размера стороны (только от толщины).
Считается один раз при импорте (`KE`).

### Модуль-уровень: сетка и граничные условия
- `nid(ix, iy)` — номер узла по координатам сетки (iy=0 — низ балки,
  ix=0 — середина пролёта);
- `EDOF (nel, 8)` — глобальные номера степеней свободы каждого элемента;
  `_IK`, `_JK` — заранее развёрнутые индексы строк/столбцов для быстрой
  сборки разреженной матрицы;
- `FIXED`/`FREE`: симметрия (ux=0 на срезе x=0) + опора (uy=0 на трёх
  нижних узлах правого края);
- `F` — вектор сил: P/2 (половина груза — половина модели!) вниз,
  размазана по 4 верхним узлам у середины пролёта;
- `PASSIVE` — маска неудаляемых клеток: площадка под грузом и пятка
  над опорой (иначе агент срежет точки приложения сил).

### `solve(density) -> (compliance, energy)`
Прямой МКЭ-решатель: собирает глобальную матрицу жёсткости
(`KE.ravel() × E_элемента` → COO → CSC), решает `K·u = F` разреженным
LU (`spsolve`) по свободным степеням свободы, возвращает:
- `compliance = Fᵀu` — податливость (чем меньше, тем жёстче структура);
- `energy (NX, NY)` — энергия деформации каждого элемента
  `½·uₑᵀ·KE·uₑ·Eₑ` — «насколько клетка работает», главный сигнал для
  резьбы.

### `von_mises(density) -> (vm, center_defl)`
Тот же решатель, но с восстановлением напряжений: в центре каждого
элемента `σ = D·B(0,0)·uₑ`, затем эквивалентное напряжение фон Мизеса
`√(σx² − σxσy + σy² + 3τ²)`. В пустых клетках обнуляется. Возвращает
также прогиб верхнего узла середины пролёта. Используется для финальной
проверки прочности готового дизайна.

### `blur(field, radius, sigma) -> field`
Гауссово сглаживание поля чувствительностей (свёртка с ядром (2r+1)²,
края — режим `edge`). Классическое лекарство от «шахматной доски» —
численного артефакта топологической оптимизации, когда клетки через одну
формально жёстки, а физически это мусор.

### `prune_hairs(d) -> d`
До трёх проходов: удаляет живые клетки, у которых меньше 2 соседей по
4-связности (кроме PASSIVE). «Волоски» не несут нагрузку и портят
печать. Удаление вершин степени 1 не может разорвать связность.

### `largest_component(density, seeds) -> density`
BFS по 4-связности от затравок (клетка под грузом и клетка над опорой);
всё, что не достижимо, обнуляется — «отвалившиеся» островки материала
не работают и не должны попадать в STL.

### `class CarveEnv`
Среда эпизодической резьбы (25 шагов на эпизод):
- `reset()` — плотность = сплошная, решается стартовый МКЭ,
  запоминается податливость сплошной балки `c0` (нормировка награды);
- `_obs()` — наблюдение: [текущая доля материала, номер шага/25];
- `step(frac, temp)` — одно «рассасывание»:
  1. чувствительности = сглаженная энергия деформации;
  2. резать можно только клетки из нижних 60% по энергии (сильно
     нагруженный «скелет» защищён жёстко — иначе случайный сэмпл
     перерезает несущий раскос и эпизод погибает);
  3. количество удаляемых = `frac` × число активных клеток, но не ниже
     целевой доли материала;
  4. вероятность удаления клетки ∝ `softmax(−энергия/temp)`:
     `temp → 0` — жадный BESO-стиль (режем строго слабейшие),
     `temp` больше — больше случайности (и «органичности»);
  5. зачистка: `largest_component` + `prune_hairs`;
  6. пересчёт МКЭ; награда только в конце эпизода:
     `−min(C/C0, 30) − 10·max(0, vol − цель − 0.02)` (кламп 30 не даёт
     развалившимся структурам взрывать градиент REINFORCE).

## topo_train.py — REINFORCE для топологической оптимизации

### `class Policy(nn.Module)`
MLP 2→32(tanh)→2: по наблюдению выдаёт средние двух действий; лог-СКО —
отдельный обучаемый параметр (стартует с −1.5, СКО клампится в
[0.03, 0.7]). `dist(obs)` возвращает `Normal(mean, std)`.

### `to_action(raw) -> (frac, temp)`
Сигмоида переводит сырые действия в физические диапазоны: доля удаления
1–7% за шаг, температура 0.03–0.6 (диапазоны подобраны по зондированию
ландшафта: выше — структуры стабильно разваливаются).

### `run_episode(policy, env, sample) -> (logp_sum, reward, info)`
Прогон одного эпизода: на каждом шаге действие сэмплируется (обучение)
или берётся среднее (детерминированная оценка); накапливается сумма
лог-вероятностей — она нужна градиенту REINFORCE.

### Основной цикл (40 обновлений × 6 эпизодов)
Батч эпизодов на одно обновление: advantage = награда эпизода минус
среднее по батчу, `loss = −mean(adv · logp)`. Батч критичен: среда
шумная (стохастическая резьба), одноэпизодный REINFORCE расходится.
Лучшая структура по всем эпизодам (включая финальный детерминированный
прогон) сохраняется в `best_topology.npy`; в консоль — ASCII-превью.

## topo_stl.py — из сетки плотностей в гладкий STL

### `cleanup(d, seeds) -> d`
То же, что `largest_component`, но для **полной** (уже зеркалённой)
сетки: BFS от груза и обеих опор, изолированные куски удаляются.

### `node_field(d) -> (nx+1, ny+1)`
Переводит клеточные плотности {0,1} в **узловое** скалярное поле:
значение узла = среднее прилегающих клеток (1–4 штук). Это дешёвое
сглаживание: одинокая клетка получает узлы ≤0.25 и исчезает на
изоуровне 0.45 (деспеклинг), ступени превращаются в фаски.

### `cell_polys(v, corners) -> [полигоны]`
Marching squares **с заливкой**: для клетки с 4 узловыми значениями
возвращает многоугольники «внутренней» части (не только изолинию):
- все углы внутри → весь квадрат; все снаружи → пусто;
- вложенная `crossing(k)` — точка пересечения изоуровня с ребром k
  (линейная интерполяция);
- обход периметра: углы внутри добавляются, на рёбрах со сменой знака
  добавляется пересечение — получается полигон в порядке CCW;
- сёдла (два противоположных угла внутри): по среднему значению клетки
  выбирается «соединить» (один полигон-перемычка) или «разъединить»
  (два треугольника) — стандартная неоднозначность marching squares.

Соседние клетки вычисляют пересечения из одних и тех же пар узловых
значений, поэтому точки на общем ребре совпадают бит-в-бит — крышки
получаются согласованными без сшивания.

### `build_organic_mesh(density) -> (tris, area)`
Полный 3D-меш экструзией:
1. 2D-триангуляция области: `cell_polys` в каждой клетке + ушная
   триангуляция (`make_stl.triangulate`) каждого полигона;
2. крышки: 2D-треугольники дублируются на z=THICK (нормали +Z, порядок
   прямой) и z=0 (нормали −Z, порядок обратный);
3. стенки: направленное ребро, у которого нет обратного двойника в
   множестве всех рёбер, — граничное; из каждого граничного ребра
   строится вертикальный прямоугольник (2 треугольника), ориентация
   наружу следует из CCW-обхода полигонов;
4. попутно считается точная площадь 2D-области (для контроля объёма).

### Блок `__main__`
1. загружает половину, считает `von_mises` → максимальное напряжение,
   прогиб, запас прочности, безопасная нагрузка;
2. зеркалит (`d[::-1]` + `d`), чистит связность;
3. строит меш, проверяет `check_mesh` (объём против `area × THICK_MM`);
4. пишет `beam_organic.stl`, печатает сводку.
