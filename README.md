# RL Projects — обучение с подкреплением на практике

Шесть проектов с Reinforcement Learning на Python: от игр (гонки, шутер)
до робототехники (многоножка в MuJoCo) и инженерного проектирования
(балка и воздушный винт с экспортом в STL для 3D-печати).

## Что такое RL — в трёх предложениях

Обучение с подкреплением (Reinforcement Learning) — это когда агент
(нейросеть или простое распределение вероятностей) выдаёт действия,
а среда возвращает награду — число «насколько хорошо получилось».
Никто не объясняет агенту, КАК решать задачу: он пробует, а алгоритм
усиливает те действия, что приносят больше награды. Всё искусство —
в конструкции награды: что поощряешь, то и получишь (включая хитрые
лазейки, которые агент обязательно найдёт).

В проектах используются два алгоритма:
- **REINFORCE** — простейший, написан вручную на PyTorch (~30 строк):
  используется в инженерных задачах (beam_rl, prop_rl);
- **PPO** — стандартный современный алгоритм из библиотеки
  stable-baselines3: используется в играх и робототехнике.

## Проекты

### 🚗 [car_racing/](car_racing/) — гонки по F1-трассе
Машинка с реалистичной физикой шин (окружность трения) учится ехать по
трассе, вдохновлённой Спа-Франкоршам, и по 30 случайным трассам.
PPO, 7 лучей-сенсоров, 13 наблюдений, дискретные действия.

```bash
cd car_racing
python main.py          # поездить самому (стрелочки)
python train.py         # обучение
python play_trained.py  # смотреть результат
```

### 🔫 [shooter/](shooter/) — self-play шутер
Дуэль «танчиков» на арене со стенами. Каждые 200k шагов противник
получает копию весов агента и становится умнее — агент учится против
самого себя (как AlphaGo). Упреждение, уклонение от пуль, фланги.

```bash
cd shooter
python train.py            # обучение (self-play)
python play.py ai bot      # смотреть модель
python play.py human bot   # сыграть самому
python play.py ai ai       # RL vs RL
```

### 🐛 [centipede/](centipede/) — робот-многоножка (MuJoCo)
Робот из 6 сегментов и 12 ног в реальных габаритах (сервоприводы MG996R)
учится ходить по командам скорости: вперёд/назад, повороты, дуги.
После обучения им можно рулить стрелочками. Цель — перенос походки на
настоящего робота (sim-to-real): шероховатый пол, фильтры действий,
только реалистичные датчики.

```bash
cd centipede
python train.py    # обучение PPO (8 параллельных симуляций)
python play.py     # рулить обученным роботом стрелочками
```

### 🏗 [beam_rl/](beam_rl/) — балка под 3D-печать
RL как инженер-конструктор: балка должна держать 40 кг при минимальной
массе. Два варианта: параметрический двутавр (REINFORCE подбирает
4 размера сечения) и органическая структура (агент «рассасывает»
ненагруженный материал, как кость; физика — метод конечных элементов).
Результат — готовые STL-файлы с проверкой прочности.

```bash
cd beam_rl
python train.py && python make_stl.py        # двутавр -> beam.stl
python topo_train.py && python topo_stl.py   # органическая -> beam_organic.stl
```

### 🌀 [prop_rl/](prop_rl/) — тихий воздушный винт
RL проектирует 13-дюймовый пропеллер: хорду и крутку лопасти в 4 сечениях
плюс обороты. Физика — классическая BEMT (blade element + momentum theory),
откалиброванная по реальному винту APC. Два режима: минимум шума при
заданной тяге и максимум тяги при лимите мощности. Результат — STL
с профилем NACA 4412, готовый к печати.

```bash
cd prop_rl
python prop_train.py && python prop_stl.py   # тихий винт -> prop_13in.stl
```

git clone <repo>

## Стек

- **Python 3.10+**
- **PyTorch** + **numpy** — нейросети и математика
- **gymnasium** — стандартный интерфейс RL-сред (бывший OpenAI Gym)
- **stable-baselines3** — готовая реализация PPO
- **pygame 2.x** — 2D-рендеринг (гонки, шутер)
- **MuJoCo** — физическая симуляция робота (многоножка)
- **scipy** — разреженные матрицы для метода конечных элементов (beam_rl)

## С чего начать читать код

Каждый проект устроен одинаково: `*_env.py` — среда (физика + награда),
`train.py` — обучение, `play*.py` / `*_stl.py` — использование результата.
Все файлы подробно прокомментированы в расчёте на читателя, который
знает программирование, но не знаком с RL. Самый простой для старта —
`beam_rl/beam_env.py` + `beam_rl/train.py` (вся суть RL в ~200 строках);
в `beam_rl/DOCS.md` есть мини-глоссарий терминов.
